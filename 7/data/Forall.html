Forall

Scala has polymorphic types (class Foo[A], type Bar[A], …) and polymorphic methods (def foo[A]), but it lacks polymorphic values. As a special case, it lacks polymorphic functions (since functions are values).

A value polymorphic in type A is a value that behaves uniformly over all types A. It is also called a universally quantified value.

The advantage of polymorphic values over polymorphic methods is that values, unlike methods, can be passed as arguments to other methods.

Scalaz encoding of polymorphic values also has the property that instantiating a polymorphic value at a specific type is a no-op at runtime. The polymorphic value itself already is a value of the specific type as well. All instantiations of a polymorphic value share the same runtime object.

Scalaz datatype Forall[F[_]] (also aliased as ∀[F[_]]) represents values polymorphic in one type variable.

Datatype Forall2[F[_, _]] (also aliased as ∀∀[F[_, _]]) represents values polymorphic in two type variables.

Polymorphic values come up naturally in many circumstances, as is illustrated by the examples below.

Typical imports

import scalaz._

Polymorphic identity function

In Scala, we can define the polymorphic identity method:

scala> def identity[A](a: A): A = a
identity: [A](a: A)A

But how would we define it as a value (of a function type) that works on any type? Let’s start by writing down its type:

scala> type IdFun[A] = A => A
defined type alias IdFun

scala> type PolyIdFun = ∀[IdFun]
<console>:16: error: not found: type ∀
       type PolyIdFun = ∀[IdFun]
                        ^

To create a value of PolyIdFun, we can use the ∀.mk method:

scala> val identity = ∀.mk[PolyIdFun].from(a => a)
<console>:15: error: not found: value ∀
       val identity = ∀.mk[PolyIdFun].from(a => a)
                      ^
<console>:15: error: not found: type PolyIdFun
       val identity = ∀.mk[PolyIdFun].from(a => a)
                           ^

The body of the implementation (here a => a) has to be typable as A => A for any type A.

Expanding the type alias PolyIdFun, the above is equivalent to

scala> val identity = ∀.mk[∀[IdFun]].from(a => a)
<console>:16: error: not found: value ∀
       val identity = ∀.mk[∀[IdFun]].from(a => a)
                      ^
<console>:16: error: not found: type ∀
       val identity = ∀.mk[∀[IdFun]].from(a => a)
                           ^

Alternatively, we can use the ∀.of method and omit the ∀ in the type parameter:

scala> val identity = ∀.of[IdFun](a => a)
<console>:16: error: not found: value ∀
       val identity = ∀.of[IdFun](a => a)
                      ^

With some help from kind-projector, we can avoid intermediate type aliases altogether:

scala> val identity = ∀.of[λ[A => A => A]](a => a)
<console>:15: error: not found: value ∀
       val identity = ∀.of[λ[A => A => A]](a => a)
                      ^

To invoke this polymorphic function, we first instantiate it at a specific type:

scala> identity[Int]
<console>:17: error: missing argument list for method identity
Unapplied methods are only converted to functions when a function type is expected.
You can make this conversion explicit by writing `identity _` or `identity(_)` instead of `identity`.
       identity[Int]
               ^

and then apply it to an argument

scala> identity[Int](5)
res1: Int = 5

Empty data structures

Consider a generic data structure, such as List[A] or Option[A]. The empty case (like Nil or None) can be the same runtime object for all types A.

The way this is achieved in the Scala standard library is to declare such data structures covariant (List[+A], Option[+A], …) and have the empty case be an instantiation of the data structure at the bottom type Nothing (object Nil extends List[Nothing]).

But covariance is not necessary (and often undesirable). These empty values are in fact polymorphic. We can define an invariant version of Option with the empty case shared across all types:

sealed trait IOption[A]
case class ISome[A](a: A) extends IOption[A]
case class INone[A] private() extends IOption[A]

object INone {
  private val instance = ∀.of[IOption](new INone())

  def apply[A](): IOption[A] = instance[A]
}

We can also create polymorphic versions of the standard empty structures:

scala> val nil: ∀[List] = ∀.of[List](Nil)
<console>:15: error: not found: type ∀
       val nil: ∀[List] = ∀.of[List](Nil)
                ^
<console>:15: error: not found: value ∀
       val nil: ∀[List] = ∀.of[List](Nil)
                          ^
scala> val emptyMap: ∀∀[Map] = ∀∀.of[Map](Map())
<console>:15: error: not found: type ∀∀
       val emptyMap: ∀∀[Map] = ∀∀.of[Map](Map())
                     ^
<console>:15: error: not found: value ∀∀
       val emptyMap: ∀∀[Map] = ∀∀.of[Map](Map())
                               ^

Natural transformation

Polymorphic functions are natural transformations. They are a generalization of the polymorphic identity function above.

     | type ~>[F[_], G[_]] = ∀[λ[A => F[A] => G[A]]]
     | 
     | val headOption: List ~> Option = ∀.mk[List ~> Option].from(_.headOption)
     | 
     | headOption[Int](List(1, 2, 3))

The above identity function is a special case of natural transformation:

     | val identity = ∀.mk[Id ~> Id].from(a => a)
     | 
     | identity.apply(5)

We can also apply a polymorphic function to a polymorphic value and obtain a polymorphic result.

     | implicit class NaturalTransformationOps[F[_], G[_]](trans: F ~> G) {
     |   def $(f: ∀[F]): ∀[G] = ∀.of[G](trans.apply.apply(f.apply))
     | }
     | 
     | headOption $ nil

Binatural transformation

Function polymorphic in two type variables.

     | type ~~>[F[_, _], G[_, _]] = ∀∀[λ[(α, β) => F[α, β] => G[α, β]]]
     | 
     | type Option2[A, B] = Option[(A, B)]
     | val pick: Map ~~> Option2 = ∀∀.mk[Map ~~> Option2].from(_.headOption)
     | 
     | pick[String, Int](Map("hi" -> 5))
     | 
     | implicit class BinaturalTransformationOps[F[_, _], G[_, _]](trans: F ~~> G) {
     |   def $(f: ∀∀[F]): ∀∀[G] = ∀∀.of[G](trans.apply.apply(f.apply))
     | }
     | 
     | val none2: ∀∀[Option2] = pick $ emptyMap

Restrictions

Since the same runtime object is shared among all instantiations of a polymorphic value, it is not a good idea to create polymorphic values of mutable data types. It even leads to unsoundness:

     | class Foo[A](var elems: List[A])
     | 
     | val foo = ∀.of[Foo](new Foo(Nil))
     | 
     | foo[Int].elems = List(1, 2, 3)

     | val s: String = foo[String].elems.head
